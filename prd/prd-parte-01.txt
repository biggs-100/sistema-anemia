PRD-001
Sistema de Seguimiento de Niños Tratados por Anemia

Versión: 1.0

Estado: Borrador Inicial

Fecha: Junio 2026

1. Resumen Ejecutivo
1.1 Descripción General

El Sistema de Seguimiento de Niños Tratados por Anemia es una aplicación de escritorio diseñada para registrar, monitorear y analizar la evolución clínica de niños diagnosticados con anemia.

El sistema permitirá almacenar información demográfica, controles clínicos, tratamientos, visitas de seguimiento y reportes estadísticos, proporcionando una herramienta confiable para establecimientos de salud, municipalidades y programas sociales.

La aplicación funcionará en red local y estará diseñada para múltiples usuarios concurrentes utilizando una base de datos SQLite centralizada con mecanismos automáticos de respaldo.

1.2 Problema Actual

Actualmente gran parte del seguimiento de pacientes se realiza mediante:

Registros físicos.
Hojas de cálculo.
Bases de datos dispersas.
Información duplicada.
Reportes manuales.

Esto genera:

Pérdida de información.
Errores de digitación.
Dificultad para realizar seguimiento.
Dificultad para generar indicadores.
Riesgo de incumplimiento de controles.
1.3 Solución Propuesta

Desarrollar una aplicación de escritorio moderna que permita:

Registro centralizado de pacientes.
Seguimiento longitudinal.
Control de tratamientos.
Generación de reportes.
Gestión de usuarios.
Auditoría de cambios.
Copias de seguridad automáticas.

Ingresar información.

Actor

6. Requerimientos Funcionales
RF-001

El sistema deberá permitir registrar pacientes.

RF-002

El sistema deberá validar documentos duplicados.

RF-003

El sistema deberá calcular automáticamente la edad.

RF-004

El sistema deberá permitir registrar múltiples controles por paciente.

RF-005

El sistema deberá almacenar resultados históricos de hemoglobina.

RF-006

El sistema deberá mostrar la evolución cronológica del paciente.

RF-007

El sistema deberá permitir registrar tratamientos.

RF-008

El sistema deberá permitir finalizar tratamientos.

RF-009

El sistema deberá generar reportes PDF.

RF-010

El sistema deberá exportar información a Excel.

RF-011

El sistema deberá permitir búsquedas rápidas.

RF-012

El sistema deberá registrar fecha y hora de cada modificación.

RF-013

El sistema deberá registrar el usuario que realizó cada cambio.

RF-014

El sistema deberá generar copias de seguridad automáticas.

RF-015

El sistema deberá restaurar respaldos anteriores.

RF-016

El sistema deberá soportar múltiples usuarios.

RF-017

El sistema deberá implementar autenticación mediante usuario y contraseña.

RF-018

El sistema deberá bloquear accesos no autorizados.

7. Requerimientos No Funcionales
RNF-001 Rendimiento

El tiempo de respuesta no deberá superar 2 segundos en operaciones normales.

RNF-002 Disponibilidad

La aplicación deberá estar disponible durante toda la jornada laboral.

RNF-003 Seguridad

Las contraseñas deberán almacenarse utilizando hash seguro.

RNF-004 Integridad

No deberá permitirse la eliminación accidental de datos relacionados.

RNF-005 Usabilidad

La interfaz deberá ser intuitiva para personal sin conocimientos técnicos.

RNF-006 Mantenibilidad

La arquitectura deberá permitir futuras ampliaciones.

RNF-007 Escalabilidad

El sistema deberá soportar crecimiento progresivo de información durante varios años.

RNF-008 Compatibilidad

Compatible con:

Windows 10
Windows 11
RNF-009 Respaldo

La aplicación deberá generar copias automáticas diarias.

RNF-010 Auditoría

Toda modificación deberá quedar registrada.
